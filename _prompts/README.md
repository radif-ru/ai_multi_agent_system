# `_prompts/` — системные промпты в markdown

Каталог содержит **системные промпты** агента в виде markdown-файлов. Полная спецификация — `_docs/prompts.md`.

## Зачем

- Системный промпт правится **без перекомпиляции**: поправил, перезапустил процесс, готово.
- Это **контент**, а не код; пусть лежит рядом с `_skills/` и `_docs/`.
- Путь главного промпта — настраивается через `AGENT_SYSTEM_PROMPT_PATH` в `.env` (по умолчанию — `_prompts/agent_system.md`); легко переключаться между вариантами / экспериментами.

## Состав

| Файл                | Кто использует                | Что делает |
|---------------------|-------------------------------|------------|
| `agent_system.md`   | `Executor` (агентный цикл)    | Главный системный промпт цикла. Объясняет модели формат ответа (JSON), набор tools, набор скиллов, правила поведения. Содержит плейсхолдеры `{{TOOLS_DESCRIPTION}}` и `{{SKILLS_DESCRIPTION}}`. |
| `summarizer.md`     | `Summarizer` (in-session + `/new`) | Системный промпт для сжатия истории в краткое резюме. Используется и при достижении `HISTORY_SUMMARY_THRESHOLD` (in-session), и при архивировании на `/new`. |

В будущих спринтах сюда добавятся `planner.md` и `critic.md` (см. `_docs/roadmap.md` Этап 4 «Multi-agent»).

## Плейсхолдеры в `agent_system.md`

Перед стартом цикла Executor заменяет эти плейсхолдеры на реальные данные:

| Плейсхолдер              | Откуда берётся                                | Формат |
|--------------------------|------------------------------------------------|--------|
| `{{TOOLS_DESCRIPTION}}`  | `ToolRegistry.list_descriptions()`             | Markdown-список вида `- name(args): description.` |
| `{{SKILLS_DESCRIPTION}}` | `SkillRegistry.list_descriptions()` (первая строка каждого `_skills/<name>/SKILL.md`) | Markdown-список вида `- name: description.` |

Подстановка — простая `str.replace`. Если плейсхолдера нет в файле — это не ошибка; подстановка пропускается. Подробности — `_docs/prompts.md` §3.

## Жизненный цикл правки

1. Открыть нужный файл в `_prompts/`.
2. Поправить.
3. Перезапустить процесс (`Ctrl+C` + `python -m app`) — изменения подхватятся.
4. Smoke-проверка в Telegram: цикл по-прежнему выходит в `final_answer` (не сломали JSON-схему ответа).
5. Если правка ломает контракт ответа модели — обновить `_docs/agent-loop.md` §2 синхронно с промптом.
6. Закоммитить `docs(prompts): ...` или `feat(prompts): ...` (если это не косметика, а смысловое изменение).

## Что НЕ кладём в `_prompts/`

- **Тело скиллов** — это `_skills/<name>/SKILL.md`, отдельная сущность (см. `_docs/skills.md`).
- **Описания tools** — они приходят из кода (`ToolRegistry.list_descriptions`); не нужны как отдельные файлы.
- **Шаблоны для отдельных команд бота** (текст `/start` или `/help`) — это UI-копи, лежит прямо в коде хендлеров (`app/adapters/telegram/handlers/commands.py`).
- **Код**.

## Многоязычность

В MVP оба промпта — на **русском** (основной пользователь общается с ботом на русском). Английский вариант — кандидат на будущий спринт (через `AGENT_SYSTEM_PROMPT_PATH=_prompts/agent_system.en.md`).
