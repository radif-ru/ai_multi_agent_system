# Команды бота

Все команды — текстовые (slash-commands Telegram). Отправляются пользователем в личный чат с ботом.

## Сводная таблица

| Команда         | Параметры         | Назначение                                                                  | В MVP |
|-----------------|-------------------|-----------------------------------------------------------------------------|:-----:|
| `/start`        | —                 | Приветствие, краткая инструкция, список команд.                             |   ✅   |
| `/help`         | —                 | Подробная справка.                                                          |   ✅   |
| `/models`       | —                 | Список доступных LLM + отметка активной.                                    |   ✅   |
| `/model`        | `<model_name>`    | Переключить активную LLM для текущего пользователя.                         |   ✅   |
| `/prompt`       | `<text>` \| —     | Установить системный промпт; без аргумента — сброс к default из `_prompts/`. |   ✅   |
| `/new`          | —                 | Архивировать сессию (саммари → чанки → `sqlite-vec`), открыть новую.        |   ✅   |
| `/reset`        | —                 | Очистить in-memory историю + сбросить per-user настройки. Архив не трогать.  |   ✅   |
| *любой текст*   | —                 | Запустить агентный цикл с этой задачей; вернуть финальный ответ.            |   ✅   |

## Подробное поведение

### `/start`

- **Вход:** нет параметров.
- **Эффект:** —
- **Ответ:**
  ```
  Привет! Я — AI-агент на локальной LLM.

  Я не просто отвечаю — я решаю задачи: думаю, выбираю инструмент,
  смотрю результат и так до финального ответа.

  Команды:
  /help — подробная справка
  /models — список моделей
  /model <имя> — выбрать модель
  /prompt <текст> — задать системный промпт (без текста — сброс)
  /new — закрыть текущую сессию (с архивированием) и начать новую
  /reset — очистить контекст и сбросить настройки
  ```

### `/help`

- **Вход:** нет параметров.
- **Эффект:** —
- **Ответ:** расширенная справка, включая текущую модель, текущий системный промпт (обрезанный до N символов), доступные tools и скиллы (имена + описания).

### `/models`

- **Вход:** нет параметров.
- **Эффект:** —
- **Ответ:** список `OLLAMA_AVAILABLE_MODELS` с пометкой активной модели пользователя.
- **Пример:**
  ```
  Доступные модели:
  • qwen3.5:4b ← активная

  Смени командой: /model <имя>
  ```

### `/model <model_name>`

- **Вход:** одно слово — имя модели.
- **Валидация:**
  - аргумент отсутствует → подсказка «Использование: `/model <имя>`, список: /models».
  - имя не в `OLLAMA_AVAILABLE_MODELS` → «Модель не найдена. Доступно: …».
- **Эффект:** записать `user_id → model_name` в `UserSettingsRegistry` (in-memory).
- **Ответ:** «Модель переключена на `<model_name>`».

### `/prompt [<text>]`

- **Вход:** произвольный текст (может быть пустым).
- **Эффект:**
  - если текст непустой → установить системный промпт пользователя (in-memory). Это **переопределение** на уровне per-user; шаблон `_prompts/agent_system.md` всё равно используется как база цикла, но `system`-сообщение в начале контекста заменяется на пользовательское.
  - если пусто → сбросить к default (читается из `AGENT_SYSTEM_PROMPT_PATH` через `PromptLoader`).
- **Ответ:** «Системный промпт обновлён.» / «Системный промпт сброшен к значению по умолчанию.»

### `/new`

- **Вход:** нет параметров.
- **Эффект:**
  1. Если `ConversationStore.get_session_log(user_id)` пуст → ничего не архивируем, просто `rotate_conversation_id(user_id)`. Ответ: «Сессия пустая, новая открыта.»
  2. Иначе вызвать `Archiver.archive(history=..., progress_callback)`, где `history` — это **полный лог сессии** (`get_session_log`), а не in-session `get_history` (см. `_docs/memory.md` §2.5; `replace_with_summary` мог сжать `get_history` до summary + tail, и ранние факты были бы потеряны):
     1. Суммировать полный лог через `Summarizer` (map-reduce для длинных сессий).
        - Логирование времени: `archive stage=summarize dur_ms=N`
     2. Разрезать саммари на чанки (`MEMORY_CHUNK_SIZE` / `MEMORY_CHUNK_OVERLAP`).
        - Логирование: `archive stage=chunking chunks=N dur_ms=N`
     3. **Параллельный embedding**: для всех чанков через `asyncio.gather` с семафором `EMBEDDING_CONCURRENCY` (default 5).
        - Логирование: `archive stage=embedding chunks=N dur_ms=N`
     4. Записать чанки + векторы + метаданные (`user_id`, `chat_id`, `conversation_id`, `chunk_index`) в `SemanticMemory`.
        - Логирование: `archive stage=database chunks=N dur_ms=N`
  3. **Прогресс**: при длительности > 5 секунд пользователю показываются этапы («⏳ Суммирую историю диалога…», «⏳ Создаю эмбеддинги для N чанков…»).
  4. Очистить `ConversationStore.clear(user_id)` и `rotate_conversation_id(user_id)`.
- **Ответ (успех):** «Архивировано N чанков, новая сессия открыта.»
- **Ответ (ошибка суммаризации/эмбеддинга):** «Архивирование не удалось: <причина>. Сессия сохранена, попробуйте `/new` ещё раз позже.» (in-memory история **не очищается**, чтобы не потерять контекст).
- **Что НЕ делает:** не трогает per-user модель / системный промпт. Это контракт: `/new` — это про **сессию**, а не про настройки.
- **Логирование:** итоговое `archive ok user_id=N conv=<id> chunks=N total_dur_ms=N`.

### `/reset`

- **Вход:** нет параметров.
- **Эффект:**
  1. Очистить in-memory историю: `ConversationStore.clear(user_id)`.
  2. Сбросить per-user настройки: `UserSettingsRegistry.reset(user_id)` (модель и системный промпт → default из `Settings` / `_prompts/`).
  3. Сгенерировать новый `conversation_id`: `rotate_conversation_id(user_id)`.
- **Ответ:** «Контекст диалога очищен, модель и системный промпт сброшены к значениям по умолчанию.»
- **Что НЕ делает:** **не пишет** ничего в архив (`sqlite-vec`). Если хочешь сохранить старую сессию — используй `/new` до `/reset` (или просто `/new`, который и так очищает in-memory).

### Произвольный текст (не команда)

- **Вход:** любое текстовое сообщение, не начинающееся с `/`.
- **Алгоритм:**
  1. Проверить длину ввода (`MAX_INPUT_LENGTH = 4000`); при превышении — подсказка «Слишком длинный запрос, сократите …» и выход.
  2. Дописать сообщение пользователя в `ConversationStore` (для контекста, в т.ч. для последующего `/new`).
  3. Вызвать `core.handle_user_task(text, user_id=..., chat_id=...)` (см. `architecture.md` §3.10).
  4. Внутри `Executor` крутится цикл `thought → action → observation → ... → final_answer`. Подробности — `agent-loop.md`.
  5. Дописать финальный ответ в `ConversationStore` как `assistant`-сообщение.
  6. **Условная in-session суммаризация**: если `len(history) >= history_summary_threshold`, вызвать `Summarizer.summarize(history[:-2])` и `replace_with_summary(..., kept_tail=2)`. Падение суммаризации → `WARNING`, ответ пользователю не страдает.
  7. Отправить ответ пользователю (`message.answer(response)`). Если ответ длиннее 4096 символов — разбивка на части (Telegram limit).
  8. При ошибке — человекочитаемое сообщение (см. `architecture.md` §7).

### Файлы (Document, Voice, Photo)

Система поддерживает три типа файлов из Telegram: документы, голосовые сообщения и фотографии.

#### Документ (Document)

- **Вход:** файл типа Document (PDF/TXT/MD).
- **Алгоритм:**
  1. Скачивает файл через `download_telegram_file` с проверкой размера (`TELEGRAM_MAX_FILE_MB`, default 20).
  2. При превышении лимита — сообщение «Файл слишком большой, отправьте файл меньшего размера».
  3. Формирует обогащённый goal: «Пользователь прислал документ {path}. Caption: {caption}. Прочитай через read_document и ответь по сути.»
  4. Передаёт goal в `core.handle_user_task` (агент использует tool `read_document`).
  5. Удаляет временный файл после обработки.
- **Поддерживаемые форматы:** PDF (через `pypdf`), TXT, MD.
- **Ограничения:** файлы читаются только из временной директории (`TMP_FILES_DIR`), защита от path traversal.

#### Голосовое сообщение (Voice/Audio)

- **Вход:** файл типа Voice или Audio (.ogg/.opus).
- **Алгоритм:**
  1. Скачивает файл через `download_telegram_file` с проверкой размера.
  2. При превышении лимита — сообщение «Файл слишком большой, отправьте файл меньшего размера».
  3. Транскрибирует через `Transcriber` (faster-whisper) → текст.
  4. Передаёт распознанный текст в `core.handle_user_task` как обычное сообщение.
  5. Удаляет временный файл после обработки.
- **Конфигурация:** `WHISPER_MODEL` (default `base`), `WHISPER_LANGUAGE` (default `ru`).
- **При недоступности faster-whisper:** сообщение «Распознавание речи недоступно, установите faster-whisper».

#### Фотография (Photo)

- **Вход:** файл типа Photo.
- **Алгоритм:**
  1. Скачивает файл через `download_telegram_file` с проверкой размера.
  2. При превышении лимита — сообщение «Файл слишком большой, отправьте файл меньшего размера».
  3. Описывает через `Vision` (Ollama vision API) → текст описания.
  4. Передаёт описание в `core.handle_user_task` как обычное сообщение.
  5. Удаляет временный файл после обработки.
- **Конфигурация:** `VISION_MODEL` (default `None`). Пример: `llava:7b` для описания изображений.
- **При недоступности vision-модели:** сообщение «Vision-модель не подключена, отправь текстом, что на картинке».

## BotFather / `setMyCommands`

При старте бота зарегистрировать команды для UI Telegram:

```python
await bot.set_my_commands([
    BotCommand(command="start",  description="Начать работу"),
    BotCommand(command="help",   description="Справка"),
    BotCommand(command="models", description="Список моделей"),
    BotCommand(command="model",  description="Выбрать модель"),
    BotCommand(command="prompt", description="Задать системный промпт"),
    BotCommand(command="new",    description="Архивировать и открыть новую сессию"),
    BotCommand(command="reset",  description="Очистить контекст и сбросить настройки"),
])
```

## Ограничения ввода

- Максимальная длина входного текста (для запроса в агентный цикл) — **4000 символов**. При превышении — сообщение «Слишком длинный запрос, сократите».
- Максимальный размер файла для скачивания из Telegram — **20 МБ** (`TELEGRAM_MAX_FILE_MB`). При превышении — сообщение «Файл слишком большой, отправьте файл меньшего размера».
- Поддерживаемые типы файлов: Document (PDF/TXT/MD), Voice/Audio (.ogg/.opus), Photo. Другие типы файлов игнорируются.
- В рамках одного цикла размер ответа модели на одном шаге ограничен `AGENT_MAX_OUTPUT_CHARS` (default 8000); при превышении — `LLMBadResponse`.

## Различие `/new` vs `/reset`

| | `/new` | `/reset` |
|--|---|---|
| Архивирует историю в `sqlite-vec` | ✅ да | ❌ нет |
| Очищает in-memory историю | ✅ да | ✅ да |
| Сбрасывает per-user модель и системный промпт | ❌ нет | ✅ да |
| Меняет `conversation_id` | ✅ да | ✅ да |
| Когда использовать | «Хочу закрыть тему, оставить след» | «Хочу всё сбросить, начать с чистого листа» |
