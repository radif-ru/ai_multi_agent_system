# Легаси и технический долг — указатель

**Это не справочник.** Содержательный перечень костылей и улучшений лежит в двух источниках:

- [`current-state.md`](./current-state.md) §2 — фактические проблемы и легаси-нюансы текущего кода (по подсекциям).

Этот файл — единая точка входа: «куда смотреть, если ищешь технический долг проекта». Содержание **не дублируется** — здесь только ссылки и краткие названия.

## Правило обновления

- **Новый костыль / баг** — записывать в `current-state.md` §2 по шаблону из `current-state.md` §5. Этот файл (`legacy.md`) трогать **не нужно**, если набор подсекций не изменился.
- **Новый кандидат на улучшение** — записывать в `roadmap.md` в подходящий этап. Этот файл трогать не нужно, если набор этапов не изменился.
- **Этот файл** обновляется только при добавлении новой подсекции `2.X` в `current-state.md` или нового этапа в `roadmap.md`. Цель — оставаться < 100 строк.
- **Закрытие** записи в `current-state.md` (перенос в §6 «История закрытий») здесь **не отражается** — указатель ссылается на актуальный список, история закрытий читается прямо в `current-state.md`.
