# `_board/` — рабочая доска проекта

Каталог содержит **процессную часть** проекта: что сейчас в работе, как устроен процесс задач, история по спринтам. Спецификация системы и состояние кода — в `_docs/` (см. `_docs/README.md`).

## Состав

| Файл | Назначение |
|------|------------|
| `plan.md` | Индекс спринтов: легенда статусов, правила работы со спринтами и задачами, шаблоны файла спринта и задачи. **Точка входа в процессную часть.** |
| `process.md` | Пошаговый алгоритм закрытия одной задачи активного спринта (`§0` подготовка → `§1`–`§11` основной цикл). Универсален, не зависит от содержания задачи. |
| `progress.txt` | Ad-hoc заметки по ходу работы: чек-листы приёмки спринтов, заметки сессий, состояние окружения. Не процесс и не спецификация. |
| `sprints/<NN>-<short-name>.md` | Файлы спринтов: цель, скоуп, AC, этапы, задачи, риски, история. Закрытые спринты остаются без правок (только запись о закрытии). |

## Порядок чтения

1. **Новый агент / разработчик:** `CLAUDE.md` (поведенческие гайдлайны) → `_docs/README.md` (спецификация) → `_board/README.md` (этот файл) → `plan.md` (индекс) → `sprints/<active>.md` (текущий спринт) → `process.md` (алгоритм).
2. **Перед взятием задачи в работу:** `process.md` §1 (что прочитать) → выбранная задача в файле активного спринта → её «Связанные документы».
3. **При создании нового спринта:** `process.md` §0.1 (открытие).
4. **При закрытии спринта:** `process.md` §0.2 (закрытие).

## `_board/` vs `_docs/`

- **`_docs/`** — *что и как устроено*: требования, архитектура, протокол агентного цикла, форматы skills/prompts, инструкции по разработке, правила тестирования. Источник истины — код, документ выравнивается под код.
- **`_board/`** — *что мы сейчас делаем*: текущий активный спринт, его задачи, процесс работы, история. Источник истины — этот каталог; код выравнивается под план.

При расхождении между документами и кодом приоритет у кода (правится `_docs/`). При расхождении между планом и фактическим состоянием задач приоритет у `_board/sprints/<active>.md` (правится план).
