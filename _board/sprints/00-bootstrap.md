# Спринт 00. Bootstrap

- **Источник:** старт проекта; ТЗ пользователя на AI-агента с локальной LLM, агентным циклом, инструментами, краткосрочной + долгосрочной (векторной) памятью, командой `/new`, скиллами и адаптером в Telegram.
- **Ветка:** `main` (инфраструктурный спринт; feature-ветка не заводится).
- **Открыт:** 2026-04-28
- **Закрыт:** —

## 1. Цель спринта

Подготовить **полноценный архитектурный фундамент** под мульти-агентную систему: документация (`_docs/`), доска (`_board/`), скелет каталогов (`app/`, `tests/`, `_skills/`, `_prompts/`, `data/`), корневые файлы (`README.md`, `.gitignore`, `.env.example`, `requirements.txt`, `pyproject.toml`).

Никакого продуктового кода не пишется — только заготовки и пустые пакеты с `__init__.py`. Логика агента, tools, памяти и handlers появятся в Спринте 01 «MVP Agent» по плану из `_docs/mvp.md` § «Что входит в MVP».

## 2. Скоуп и non-goals

### В скоупе

- Установка Python 3.11+, Ollama, выбор моделей (`qwen3.5:4b` для LLM, `nomic-embed-text` для эмбеддингов).
- Создание `.gitignore`, `.env.example`, `requirements.txt`, `pyproject.toml`, `README.md`.
- Полная документация `_docs/` (17 файлов: README, mvp, requirements, architecture, agent-loop, memory, tools, skills, prompts, stack, instructions, project-structure, commands, testing, roadmap, current-state, legacy, links).
- Полная доска `_board/`: `README.md`, `plan.md`, `process.md`, `progress.txt`, `sprints/00-bootstrap.md` (этот файл), `sprints/01-mvp-agent.md` (план следующего спринта).
- Создание `_skills/` со схемой и одним примером (`_skills/example-summary/SKILL.md`).
- Создание `_prompts/` с системным промптом агента (`agent_system.md`, переведён на русский и улучшен относительно ТЗ) и промптом суммаризации (`summarizer.md`).
- Скелет `app/` со всеми пустыми пакетами и `__init__.py`.
- Скелет `tests/` с `conftest.py` и пустыми пакетами.

### Вне скоупа (non-goals)

- Реализация конфигурации (`Settings`), логирования, LLM-клиента, агентного цикла, tools, памяти, handlers — всё это в Спринте 01.
- Реальный запуск бота, прогон агентного цикла, ручная проверка `/new` — это в Спринте 01.
- Создание тестов на код — кода нет, тестировать нечего; в этом спринте присутствуют только инфраструктурные проверки (импортируется ли пакет).

## 4. Решения по архитектуре, принятые в этом спринте

Зафиксировано здесь, чтобы при ретроспективе не искать по логам диалогов. Подробное обоснование — в указанных документах.

| Решение | Ссылка на документ | Краткое обоснование |
|---------|---------------------|---------------------|
| **Векторная БД — `sqlite-vec`** (а не FAISS / chromadb / sqlite-vss) | `_docs/memory.md` §3.1 | Один `.db`-файл на всё; pure-C extension; ставится через `pip install`; преемник sqlite-vss; легко мокается в тестах. |
| **LLM по умолчанию — `qwen3.5:4b`** (как в ТЗ); единственная модель в `OLLAMA_AVAILABLE_MODELS` (можно расширить через `.env` без правки кода) | `_docs/stack.md` §3 | Ровно из ТЗ; разумный размер для агентного цикла на CPU. |
| **Embedding — `nomic-embed-text`** (768d) | `_docs/stack.md` §3, `_docs/memory.md` §3.2 | Хорошо работает с русским; 768d удобно ложится в `sqlite-vec`. Меняется через `.env`. |
| **Telegram-режим — long polling** | `_docs/architecture.md` §2, `_docs/requirements.md` CON-4 | Не нужен публичный URL / TLS; webhook отложен в roadmap (Этап 6). |
| **Архитектура single-process с изоляцией слоёв под мульти-агент** | `_docs/architecture.md` §2, §7 | Сейчас один Executor, но `core/orchestrator.py` — единственная точка входа от любого адаптера; добавление Planner / Critic в будущем не ломает tools / memory / адаптер. |
| **Skills как markdown-файлы**: `_skills/<name>/SKILL.md`, первая строка `Description: ...` инжектится в системный промпт; полное тело подгружается tool'ом `load_skill` | `_docs/skills.md` | Расширяемость без перекомпиляции; экономия контекста слабых LLM (полное тело скилла подгружается только когда нужно). |
| **Prompts как markdown-файлы**: `_prompts/agent_system.md`, `_prompts/summarizer.md`; путь главного — через `AGENT_SYSTEM_PROMPT_PATH` в `.env` | `_docs/prompts.md` | Правка без перекомпиляции; легко переключаться между вариантами промптов. |
| **Команда `/new` пишет в архив, `/reset` — нет** | `_docs/commands.md` § «Различие /new vs /reset» | Семантически разные действия: `/new` — «закрой тему, оставь след», `/reset` — «забудь и сбрось настройки». |
| **В архив — только саммари, не сырая история** | `_docs/memory.md` §1, `_docs/requirements.md` CON-1 | Приватность + малый объём. |
| **Документация и сообщения коммитов — на русском** | `_docs/instructions.md` §1, `_docs/README.md` § «Язык документации» | Запрос пользователя; технические идентификаторы — латиницей. |
