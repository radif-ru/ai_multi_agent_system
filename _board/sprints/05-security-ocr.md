# Спринт 05. Усиление безопасности и OCR-рефакторинг

- **Источник:** ТЗ "Взломай и защити AI-агента" + запрос пользователя (расширение OCR за пределы PDF)
- **Ветка:** `feature/05-security-ocr` (от `main`; см. `process.md` §2, п.2).
- **Открыт:** 2026-05-06
- **Закрыт:** —

## 1. Цель спринта

Спринт объединяет две независимые, но важные задачи: (1) внедрение мер защиты от типичных атак на LLM-системы (Prompt Injection, Data Leakage, Jailbreak, Tools Injection) на основе рекомендаций из ТЗ по безопасности, и (2) рефакторинг OCR-логики в самостоятельный сервис с добавлением tool'а для одиночных изображений. Обе задачи не затрагивают агентный цикл, событийную модель или модуль Users — спринт сознательно ограничен улучшением существующих подсистем.

## 2. Скоуп и non-goals

### В скоупе

**Безопасность:**
- Модуль `app/security/` с `InputSanitizer` для фильтрации prompt injection паттернов
- Интеграция `InputSanitizer` в Telegram-хендлеры
- Маскирование путей к файлам в goal через `FileIdMapper`
- Обновление system prompt с инструкциями о безопасности
- Добавление allowlist для опасных tools (http_request, read_file, read_document)
- Валидация ответа модели на наличие системной информации
- Документация: новый `_docs/security.md` с описанием мер защиты

**OCR:**
- Новый сервис `app/services/ocr.py` с чистым контрактом распознавания текста с изображений
- Рефакторинг `app/tools/read_document.py::_read_pdf` — делегирование OCR в сервис
- Новый tool `app/tools/ocr_image.py` для произвольных изображений в `tmp/`
- Новый skill `_skills/ocr/SKILL.md` с правилами выбора OCR vs Vision
- Обновление конфигурации (`app/config.py`): вынесение `ocr_default_lang`, `ocr_min_text_threshold`
- Обновление документации: `_docs/tools.md`, `_docs/skills.md`, `_docs/architecture.md`

### Вне скоупа (non-goals)

**Безопасность:**
- Полноценный AI-guard (второй LLM как фильтр) — бонусная задача
- Rate limiting — не применимо к stateless боту
- Авторизация и аутентификация — не этот спринт
- Интеграция внешних инструментов безопасности (Garak, PyRIT, Burp Suite)

**OCR:**
- Замена `pytesseract` на другой OCR-движок (PaddleOCR, EasyOCR, облачные API)
- Авто-OCR на upload-pipeline фотографий (прозрачное подмешивание текста к описанию)
- Переход на хеш-кеш OCR вместо файлового
- Многостраничные не-PDF документы (TIFF и т. п.)
- Любые изменения в `DescribeImageTool` (vision) и в Vision-сервисе

**Общее:**
- Изменения событийной шины и модуля Users — это спринт 04
- Изменение агентного цикла или контракта tools
